## 用法示例

### ChatGPT / Claude / Gemini 的 Web UI

直接把 URL 貼進對話即可：

```txt
請基於這份文件回答我後續的問題：
https://anibt.net/llms-full.en.txt
```

模型會拉取並把整份文件進上下文。`llms-full.en.txt` ~80 KB，對主流大模型來說一次喂完沒問題。

### Cursor / Continue / Cline 等 IDE Agent

把 `.md` 端點配成"自定義文件"。Cursor 的 `@Docs` 也吃這種格式：

```txt
https://anibt.net/llms.txt
```

它會自動展開索引、按需抓取每篇 `.md`。

### RAG / 自建 pipeline

```python title="拉取整站做向量化"
import requests, re

base = 'https://anibt.net'
index = requests.get(f'{base}/llms.txt').text

# llms.txt 裡每行的 (link) 就是一篇文件；加上 .md 字尾就是源
urls = re.findall(r'\(([^)]+)\)', index)
for path in urls:
    if not path.startswith('/docs'):
        continue
    md = requests.get(f'{base}/zh-CN{path}.md').text
    # → 喂進你的 chunker / embedder
```

如果你只關心英文，把語言段換成 `/en` 即可（預設 zh-CN 不帶字首，英文是 `/en`，繁體是 `/zh-Hant`）。

## 端點對比

| 端點                    | 內容                            | 推薦用途                          |
| ----------------------- | ------------------------------- | --------------------------------- |
| `/llms.txt`             | 站點目錄 + 連結（數 KB）         | 給 Agent 當 sitemap，按需抓取     |
| `/llms-full.txt`        | 全部 zh-CN 內容（約 80 KB）     | 一次性喂中文 LLM                   |
| `/llms-full.en.txt`     | 全部英文內容                    | 喂國際 LLM / 寫英文回答           |
| `/<lang>/docs/<slug>.md`| 單頁源 Markdown                | 精準引用某一篇                    |

<Callout type="info">
  這些端點都是**純靜態產物**，快取 1 年（`Cache-Control: s-maxage=31536000`）。Wiki 更新發布後會重新構建，幾分鐘內端點跟著更新。
</Callout>

## 內容覆蓋

`/llms.txt` 與 `/llms-full.*` 的來源就是你正在看的這份 Wiki：

- 站點同步流程
- [公開 API](/docs/open-api)（RSS / JSON / 快取 / 限流 / 故障排查）
- [Tracker 地址](/docs/tracker)
- [相關專案](/docs/projects)

任何一篇 Wiki 頁面都可以加 `.md` 拿到源。例如：

```txt
原始頁面     https://anibt.net/zh-CN/docs/open-api/rss-anime
Markdown 源  https://anibt.net/zh-CN/docs/open-api/rss-anime.md
```
