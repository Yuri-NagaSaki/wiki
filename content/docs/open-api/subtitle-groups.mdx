title: 字幕组目录
description: GET /api/subtitle-groups —— 公开字幕组列表，第三方应用拼 RSS 前先拉这一份做 name → slug 映射。
---

`/api/subtitle-groups` 返回站内所有可见字幕组的 **slug、展示名、发布数与番剧数**。第三方应用应在启动时调用一次，把响应缓存到本地，再走 RSS。

<Callout type="info">
  数据极少变化，几乎全部命中 304。配合 `If-None-Match` 轮询带宽接近零。
</Callout>

## 请求

```http title="HTTP"
GET https://site.anibt.net/api/subtitle-groups
Accept: application/json
```

## 304 支持

ETag 形如 `W/"sg{count}-{maxCreatedAt}-{releaseSum}-{urlHash}"`。再次请求带 `If-None-Match` 即拿 304。

```bash title="验证 304"
H=$(curl -sI "https://site.anibt.net/api/subtitle-groups")
ETAG=$(echo "$H" | grep -i '^etag:' | cut -d' ' -f2- | tr -d '\r')
curl -sI -H "If-None-Match: $ETAG" "https://site.anibt.net/api/subtitle-groups" | head -1
# → HTTP/2 304
```

## 实战：用 jq 挑出目标字幕组

```bash title="挑 4 个组"
curl -s https://site.anibt.net/api/subtitle-groups \
  | jq '.data[] | select(.name | test("Kirara Fantasia|Prejudice-Studio|北宇治字幕组|拨雪寻春")) | {name, slug}'
```

输出：

```json title="结果"
{"name":"Kirara Fantasia","slug":"kirara-fantasia"}
{"name":"Prejudice-Studio","slug":"pre-s"}
{"name":"北宇治字幕组","slug":"kitauji-sub"}
{"name":"拨雪寻春","slug":"haruhana"}
```

## 缓存与限流

| 项目         | 值                                                        |
| ------------ | --------------------------------------------------------- |
| `Cache-Control` | `public, max-age=300, s-maxage=300, stale-while-revalidate=86400` |
| 速率限制     | 与 RSS 端点共用 60 秒窗口，详见 [限流与上限](/docs/open-api/limits) |
| 条件请求     | 支持 `If-None-Match`、`If-Modified-Since`                  |
