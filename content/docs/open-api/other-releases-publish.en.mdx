---
title: Publish other releases
description: POST /api/other-releases/publish — publish manga, music, RAW, and stage releases with a group API key; no bgmId required.
---

`/api/other-releases/publish` is the write endpoint for **non-anime releases** from a release group. Use it for manga, music, RAW, and stage releases. It uses the same group API key system, does not require `bgmId`, and stays separate from anime releases.

<Callout type="warn">
  This is a **write endpoint**. A leaked key lets others publish under your group name, so use it only from server-side scripts or CI secrets.
</Callout>

## URL

```http title="HTTP"
POST https://site.anibt.net/api/other-releases/publish
Authorization: Bearer <YOUR_API_KEY>
```

`X-API-Key: <YOUR_API_KEY>` is also accepted, but `Authorization: Bearer` is recommended for consistency.

## Two request shapes

<Tabs items={['multipart/form-data (recommended)', 'application/json']}>
  <Tab value="multipart/form-data (recommended)">
    Upload a `.torrent` file. The backend extracts the magnet, title, file size, and trackers automatically. Sending `category` and `title` explicitly is still recommended.

    ```bash title="Upload an other release torrent"
    curl -X POST 'https://site.anibt.net/api/other-releases/publish' \
      -H 'Authorization: Bearer <YOUR_API_KEY>' \
      -F 'category=manga' \
      -F 'title=Example Manga Vol.01' \
      -F 'customTags=digital,english' \
      -F 'notes=Release notes support Markdown.' \
      -F 'torrent=@/path/to/release.torrent'
    ```

    `customTags` / `tags` are comma-separated in forms. `trackers` may be newline-separated or a JSON array string.
  </Tab>
  <Tab value="application/json">
    Send a magnet directly when your automation already has one and does not upload a torrent file.

    ```bash title="JSON body"
    curl -X POST 'https://site.anibt.net/api/other-releases/publish' \
      -H 'Authorization: Bearer <YOUR_API_KEY>' \
      -H 'Content-Type: application/json' \
      --data @- <<'JSON'
    {
      "category": "music",
      "title": "Example Album FLAC",
      "magnetLink": "magnet:?xt=urn:btih:...",
      "fileSize": 734003200,
      "customTags": ["FLAC", "BK"],
      "notes": "**Lossless audio**, BK included."
    }
    JSON
    ```

    `magnetBase64` is also accepted; it can contain either the raw magnet or a Base64-encoded magnet.
  </Tab>
</Tabs>

<Callout type="info">
  `category` and `title` are required. At least one of `torrent`, `torrentStorageId`, `magnetLink`, or `magnetBase64` must be provided.
</Callout>

## Response

```json title="Success response"
{
  "ok": true,
  "result": {
    "releaseId": "orel_DYXiEZs5KMep",
    "groupId": "org_xxx",
    "category": "manga",
    "createdTorrentFileKey": "torrent/..."
  }
}
```

Build the detail page from `releaseId`: `https://anibt.net/other-release/<releaseId>`. After publishing, the matching category RSS will include the item. See [Other release RSS](/docs/open-api/rss-other).

## Common errors

| HTTP | Cause                                                   | Fix                                                           |
| ---- | ------------------------------------------------------- | ------------------------------------------------------------- |
| 400  | Missing `category` / `title`, or no torrent / magnet source | Fill required fields; ensure `category` is one of the four allowed values |
| 401  | API key missing                                          | Add `Authorization: Bearer <YOUR_API_KEY>`                    |
| 403  | API key invalid / revoked, or the user is no longer in the group | Generate a new key and confirm group membership               |
| 404  | `torrentStorageId` points to a missing file              | Upload with multipart, or verify the file key belongs to the group |
| 429  | Write rate too high                                      | Back off according to the response headers and error message  |
