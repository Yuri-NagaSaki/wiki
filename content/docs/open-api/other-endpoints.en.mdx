## Group short link `/rss/group/<slug>.xml`

Equivalent to `/rss/anime.xml?groupSlug=<slug>`, but with a shorter URL — handy for READMEs and forums.

<Callout type="warn">
  The path **must** end with `.xml`. `?limit=` is supported, but tag filters are **not**; for filtering, use [the main entry](/docs/open-api/rss-anime).
</Callout>

```txt title="Examples"
https://anibt.net/rss/group/kirara-fantasia.xml
https://anibt.net/rss/group/pre-s.xml
https://anibt.net/rss/group/kitauji-sub.xml
https://anibt.net/rss/group/haruhana.xml
```

## Site-wide magnets `/rss/magnets.xml`

Good for "what's new across the whole site".

```txt title="Common queries"
https://anibt.net/rss/magnets.xml
https://anibt.net/rss/magnets.xml?limit=200
https://anibt.net/rss/magnets.xml?q=葬送的芙莉莲
https://anibt.net/rss/magnets.xml?resolution=1080p&language=CHS
```

<Callout type="info">
  Simple queries cache for **120s**; complex queries (with `q` and multiple filters) cache for **60s**. Both layer on `stale-while-revalidate=600s`, so the CDN serves stale content immediately and refreshes asynchronously. See [HTTP caching & 304](/docs/open-api/caching).
</Callout>

## Seasonal anime `/api/seasons/anime`

Used by external trackers, dashboards, and editor helpers. **Returns the current-season anime list and the available seasons.**

### Request

```txt title="Several call shapes"
GET https://site.anibt.net/api/seasons/anime
GET https://site.anibt.net/api/seasons/anime?season=2026春
GET https://site.anibt.net/api/seasons/anime?season=2026-SPRING   # ISO form also accepted
GET https://site.anibt.net/api/seasons/anime?bgmId=543360         # exact-match a single show
```

### Caching & rate limiting

| Item              | Value                                                                                |
| ----------------- | ------------------------------------------------------------------------------------ |
| Rate limit        | **30 requests / 60 s / IP** (shared with `/api/anime/groups`)                         |
| `Cache-Control`   | `max-age=600, s-maxage=600, stale-while-revalidate=86400` (**10 min fresh + 24 h SWR**) |
| Conditional req.  | `If-None-Match` (ETag like `W/"sa-{year}-{SEASON}[-b{bgmId}]-{count}-{lastMs}-{urlHash}"`), `If-Modified-Since` |

```bash title="Pull and 304"
curl 'https://site.anibt.net/api/seasons/anime'
curl 'https://site.anibt.net/api/seasons/anime?season=2026春'
curl 'https://site.anibt.net/api/seasons/anime?bgmId=543360'

ETAG=$(curl -sSI 'https://site.anibt.net/api/seasons/anime' | grep -i ^etag: | awk '{print $2}' | tr -d '\r')
curl -i 'https://site.anibt.net/api/seasons/anime' -H "If-None-Match: $ETAG"
# → HTTP/2 304
```

## Per-show groups `/api/anime/groups`

> "Who's working on this anime?" Pass a single `bgmId` and get every active group, the latest update time, and **up to 30 recent releases**.

Designed to feed `groupSlug` into `/rss/anime.xml`, and equally useful for building "show detail / release list" UIs directly.

### Request

```txt
GET https://site.anibt.net/api/anime/groups?bgmId=543360
```

<TypeTable
  type={{
    'bgmId': {
      description: 'Bangumi id (positive integer), e.g. 543360.',
      type: 'integer',
    },
  }}
/>

### Response (excerpt)

```jsonc title="200 OK"
{
  "ok": true,
  "ts": 1776841967132,
  "data": {
    "bgmId": 543360,
    "animeId": "nh73r0ernee7s0afv9s82f763983gcz7",
    "title": { /* primary / chinese / romaji / english / japanese */ },
    "groups": [
      {
        "groupId": "org_aagmgM9i0ZyR",
        "slug": "kitauji-sub",
        "name": "北宇治字幕组",
        "status": "ACTIVE",
        "lastUpdatedAt": 1776624843260,
        "items": [
          {
            "releaseId": "rel_fItEHE7r1gR1",
            "title": "[北宇治字幕组] 上伊那牡丹，酒醉身姿似百合 [02][WebRip][HEVC_AAC][繁日內嵌]",
            "magnet": "magnet:?xt=urn:btih:282e2b…",
            "episodeKey": "2",
            "resolution": "1080p",
            "language": ["CHT", "JP"],
            "subtitle": "EMBEDDED",
            "publishedAt": 1776624843260
          }
          /* … up to 30 entries, sorted by publishedAt desc … */
        ]
      }
      /* … other groups … */
    ],
    "total": 4
  }
}
```

- `ts`: same as `/api/seasons/anime` — wall-clock time, useful for verifying CDN caching.
- The list is sorted by `lastUpdatedAt` desc (most recently active first).
- `items`: up to 30 latest releases for that group on that show, sorted by `publishedAt` desc. You can render a "show × group" resource panel directly without pulling RSS.
  - `magnet` may be `null` if only a torrent file exists.
  - `episodeKey` is a stable episode key (e.g. `01`, `SP01`) suitable for sorting / aggregation.
  - `language`: `("CHS" | "CHT" | "JP" | "EN")[]`.
  - `subtitle`: `"EMBEDDED" | "EXTERNAL" | "BUILT_IN" | "NONE"`.
- `slug` matches [the catalogue](/docs/open-api/subtitle-groups) exactly and can be fed to `/rss/anime.xml?groupSlug=...`.
- An unlisted or hidden id returns `404 ANIME_NOT_FOUND`.

### Typical use — build an RSS for a tracker

```bash title="Two-step pipeline"
# Step 1: given a bgmId, fetch the group list
curl -s 'https://site.anibt.net/api/anime/groups?bgmId=543360' \
  | jq -r '.data.groups[].slug'
# kitauji-sub
# kirara-fantasia
# pre-s
# haruhana

# Step 2: build an RSS (multiple groups, CHS + 1080p)
curl 'https://anibt.net/rss/anime.xml?bgmId=543360&groupSlug=kitauji-sub,kirara-fantasia,pre-s,haruhana&language=CHS&resolution=1080p&limit=50'
```

### Caching & rate limiting

| Item              | Value                                                                              |
| ----------------- | ---------------------------------------------------------------------------------- |
| Rate limit        | **30 requests / 60 s / IP** (shared with `/api/seasons/anime`)                      |
| `Cache-Control`   | `max-age=300, s-maxage=300, stale-while-revalidate=3600` (**5 min fresh + 1 h SWR**) |
| Conditional req.  | `If-None-Match` (`W/"ag-{bgmId}-{count}-{lastMs}-{urlHash}"`), `If-Modified-Since`  |

<Callout type="info">
  When a group publishes a new release, `lastUpdatedAt` advances, the ETag changes, and the cache invalidates naturally. Polling intervals shorter than 5 minutes give you no extra benefit.
</Callout>
