---
title: Inspect your group
description: GET /api/subtitle-groups/me — authenticated lookup of the calling group's basic info and stats.
---

`/api/subtitle-groups/me` is the **authenticated read** endpoint. With the group's API key, it returns the group's basic info (name, slug, description, website) and stats (releases, anime coverage, members).

Use it for: a CI banner that prints which group key is in use, a dashboard widget showing live group state, or a sanity check ("am I authenticated as the right group?") in automation.

## Request

```http title="HTTP"
GET https://site.anibt.net/api/subtitle-groups/me
Authorization: Bearer <YOUR_API_KEY>
```

```bash title="curl"
curl -H "Authorization: Bearer <YOUR_API_KEY>" \
  https://site.anibt.net/api/subtitle-groups/me
```

## Response

```jsonc title="200 OK"
{
  "ok": true,
  "data": {
    "_id": "...",
    "name": "My Group",
    "slug": "my-group",
    "description": "...",
    "website": "https://...",
    "stats": {
      "totalReleases": 42,
      "totalAnimes": 10,
      "memberCount": 5
    },
    "status": "active",
    "createdAt": 1700000000000
  }
}
```

## Relationship to the public catalogue

<Callout type="info">
  Key differences from [`/api/subtitle-groups`](/docs/open-api/subtitle-groups):
  - `/api/subtitle-groups`: **public**, returns the trimmed list of **all** groups, no auth.
  - `/api/subtitle-groups/me`: **authenticated**, returns detailed info for the **caller's** group only — adds `stats`, internal `_id`, `description`, and `status`.
</Callout>

## Errors

| HTTP | Cause                                              | Fix                                  |
| ---- | -------------------------------------------------- | ------------------------------------ |
| 401  | Missing or malformed `Authorization` header        | Use `Bearer ` prefix; mind the space |
| 403  | API key invalid / revoked                          | Re-issue from the group dashboard    |
