---
title: Bangumi search
description: GET /api/bgm/search — public Bangumi subject lookup. Supports Chinese, English, Japanese, and romaji keywords.
---

`/api/bgm/search` is a **public** endpoint with no auth required. Use it to map "the title a user typed" to a stable Bangumi.tv subject id (`bgmId`) before passing it to [`/rss/anime.xml`](/docs/open-api/rss-anime) or [`/api/releases/publish`](/docs/open-api/releases-publish).

## Request

```txt title="URL"
GET https://site.anibt.net/api/bgm/search?q=<keyword>&limit=<n>
```

Any of Chinese, English, Japanese, or romaji works.

```bash title="curl"
curl -s -G 'https://site.anibt.net/api/bgm/search' \
  --data-urlencode 'q=葬送的芙莉莲' -d 'limit=3'
```

## Practical: search → build an RSS URL

```bash title="One-liner"
BGM=$(curl -s -G 'https://site.anibt.net/api/bgm/search' \
  --data-urlencode 'q=Frieren' -d 'limit=1' \
  | jq -r '.data[0].bgmId')

echo "https://anibt.net/rss/anime.xml?bgmId=$BGM&resolution=1080p&language=CHS"
```

## Notes

<Callout type="info">
  Results are sorted by Bangumi's internal relevance, **not** popularity. For ambiguous titles (multiple seasons, identical names, etc.) raise `limit` and let the user pick.
</Callout>

<Callout type="warn">
  This endpoint pass-throughs upstream Bangumi data, so result quality depends on Bangumi itself. In production code, cache the resolved `bgmId` locally — don't query on every request.
</Callout>
