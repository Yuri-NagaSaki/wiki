`/api/subtitle-groups` 返回站內所有可見字幕組的 **slug、展示名、釋出數與番劇數**。第三方應用應在啟動時呼叫一次，把響應快取到本地，再走 RSS。

<Callout type="info">
  資料極少變化，幾乎全部命中 304。配合 `If-None-Match` 輪詢頻寬接近零。
</Callout>

## 請求

```http title="HTTP"
GET https://site.anibt.net/api/subtitle-groups
Accept: application/json
```

## 304 支援

ETag 形如 `W/"sg{count}-{maxCreatedAt}-{releaseSum}-{urlHash}"`。再次請求帶 `If-None-Match` 即拿 304。

```bash title="驗證 304"
H=$(curl -sI "https://site.anibt.net/api/subtitle-groups")
ETAG=$(echo "$H" | grep -i '^etag:' | cut -d' ' -f2- | tr -d '\r')
curl -sI -H "If-None-Match: $ETAG" "https://site.anibt.net/api/subtitle-groups" | head -1
# → HTTP/2 304
```

## 實戰：用 jq 挑出目標字幕組

```bash title="挑 4 個組"
curl -s https://site.anibt.net/api/subtitle-groups \
  | jq '.data[] | select(.name | test("Kirara Fantasia|Prejudice-Studio|北宇治字幕組|撥雪尋春")) | {name, slug}'
```

輸出：

```json title="結果"
{"name":"Kirara Fantasia","slug":"kirara-fantasia"}
{"name":"Prejudice-Studio","slug":"pre-s"}
{"name":"北宇治字幕組","slug":"kitauji-sub"}
{"name":"撥雪尋春","slug":"haruhana"}
```

## 快取與限流

| 專案         | 值                                                        |
| ------------ | --------------------------------------------------------- |
| `Cache-Control` | `public, max-age=300, s-maxage=300, stale-while-revalidate=86400` |
| 速率限制     | 與 RSS 端點共用 60 秒視窗，詳見 [限流與上限](/docs/open-api/limits) |
| 條件請求     | 支援 `If-None-Match`、`If-Modified-Since`                  |
