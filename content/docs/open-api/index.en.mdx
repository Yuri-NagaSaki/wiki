AniBT exposes public data through two kinds of endpoints: **RSS** for subscriptions and **JSON APIs** for programmatic integration. Every endpoint is anonymously readable, and they all support **HTTP 304 Not Modified**, so polling can be frequent without cost.

<Callout type="info">
  Standard RSS clients like qBittorrent, Sonarr, FlexGet, and Tiny Tiny RSS work **without any extra configuration** — caching and conditional requests are emitted in the response headers automatically.
</Callout>

## Design overview

The system is layered by responsibility so third parties can pick the right endpoint quickly:

```txt title="Data flow"
┌────────────────────────────────────────────────────────────┐
│            Third-party client (downloader / bot)             │
└────────────────────────────────────────────────────────────┘
        │ ① At startup / once a day: fetch the group catalogue
        ▼
┌────────────────────────────────────────────────────────────┐
│  GET https://site.anibt.net/api/subtitle-groups              │
│  Returns [{slug, name, totalReleases, totalAnimes, ...}]     │
│  → Map "display name" or "user input" to a stable slug       │
└────────────────────────────────────────────────────────────┘
        │ ② When subscribing / every N minutes: pull RSS
        ▼
┌────────────────────────────────────────────────────────────┐
│  /rss/anime.xml?bgmId=...&groupSlug=...&resolution=...      │  Main entry (most flexible)
│  /rss/group/<slug>.xml                                      │  Group short link
│  /rss/magnets.xml                                           │  Site-wide latest
│  Returns RSS 2.0 + BEP 36 (enclosure + <torrent infoHash>)  │
└────────────────────────────────────────────────────────────┘
        │ ③ Send the previous ETag on every poll
        ▼
┌────────────────────────────────────────────────────────────┐
│  Server uses a fast weak-ETag comparison:                    │
│   - No change   → 304 Not Modified (empty body, ~tens B)     │
│   - Changed     → 200 + full RSS                             │
└────────────────────────────────────────────────────────────┘
```

## Stability guarantees

- Paths and query parameters are part of the public contract — **no breaking changes**; new parameters are additive only.
- `slug` values are stable once published.
- Internal primary keys (e.g. `organizationId`) are not part of the public contract; always use `slug`.
