---
title: 釋出資源
description: POST /api/releases/publish —— 字幕組用 API Key 釋出種子的寫入端點；支援 JSON（磁力）和 multipart/form-data（種子檔案）兩種格式。
---

`/api/releases/publish` 是字幕組**寫入**資源的唯一端點。需要在請求頭裡攜帶字幕組 API Key 鑑權；支援 `application/json`（直接傳磁力連結）和 `multipart/form-data`（上傳種子檔案）兩種格式。

<Callout type="warn">
  這是**寫入介面**，呼叫頻率與配額由字幕組維度獨立計數；與公開 RSS 的 IP 速率視窗不共享。Key 一旦洩露請立即在後臺重置。
</Callout>

## URL

```http title="HTTP"
POST https://site.anibt.net/api/releases/publish
Authorization: Bearer <YOUR_API_KEY>
```

## 兩種請求格式

<Tabs items={['multipart/form-data（推薦）', 'application/json']}>
  <Tab value="multipart/form-data（推薦）">
    上傳種子檔案，後端**自動從種子裡提取**磁力連結、infoHash、檔案大小等後設資料。

    ```bash title="上傳種子"
    curl -X POST 'https://site.anibt.net/api/releases/publish' \
      -H 'Authorization: Bearer <YOUR_API_KEY>' \
      -F 'bgmId=400602' \
      -F 'title=[字幕組] 葬送的芙莉蓮 - 01 [1080p].mkv' \
      -F 'episodeKey=01' \
      -F 'resolution=1080p' \
      -F 'language=CHS' -F 'language=JP' \
      -F 'subtitle=EXTERNAL' \
      -F 'format=MKV' \
      -F 'torrent=@/path/to/release.torrent'
    ```

    `language` 重複表示多個值（CHS + JP）。
  </Tab>
  <Tab value="application/json">
    直接給磁力連結（**仍然需要先有種子**，因為後端只接受能解析的合法 magnet）。

    ```bash title="JSON 請求"
    curl -X POST 'https://site.anibt.net/api/releases/publish' \
      -H 'Authorization: Bearer <YOUR_API_KEY>' \
      -H 'Content-Type: application/json' \
      --data @- <<'JSON'
    {
      "bgmId": 400602,
      "title": "[字幕組] 葬送的芙莉蓮 - 01 [1080p].mkv",
      "magnetBase64": "magnet:?xt=urn:btih:...",
      "episodeKey": "01",
      "resolution": "1080p",
      "language": ["CHS", "JP"],
      "subtitle": "EXTERNAL",
      "format": "MKV",
      "fileSize": 734003200,
      "notes": "**字型內嵌**，缺幀問題已修復。"
    }
    JSON
    ```

    `magnetBase64` 既接受原始 magnet 連結，也接受 Base64 編碼後的磁力。
  </Tab>
</Tabs>

## 預覽模式

釋出前想先看看渲染效果（標題解析、Markdown 備註、磁力是否正確），把 `preview=true` 即可：

```bash title="預覽，不公開"
curl -X POST 'https://site.anibt.net/api/releases/publish' \
  -H 'Authorization: Bearer <YOUR_API_KEY>' \
  -F 'bgmId=400602' \
  -F 'title=[字幕組] 葬送的芙莉蓮 - 01 [1080p].mkv' \
  -F 'preview=true' \
  -F 'torrent=@/path/to/release.torrent'
```

返回的預覽 URL **10 分鐘後過期**，**不會**進入 RSS / 搜尋 / 前端列表。適合自動化流水線在正式釋出前做最後一次人工確認。

## 鑑權與 Key 管理

<Steps>
  <Step>
    ### 在字幕組後臺生成 API Key
    每個字幕組可獨立生成多個 Key，建議為 CI / 個人指令碼各申請一把。
  </Step>
  <Step>
    ### 在請求頭攜帶
    ```http
    Authorization: Bearer <YOUR_API_KEY>
    ```
  </Step>
  <Step>
    ### Key 洩露立即重置
    舊 Key 立即失效；新 Key 只在生成時顯示一次，請妥善儲存。
  </Step>
</Steps>

<Callout type="warn">
  Key 不要寫進客戶端程式碼、不要 commit 到倉庫、不要貼 issue。CI 必須用 secret 注入。
</Callout>

## 常見錯誤

| HTTP | 原因                                              | 處理                                                          |
| ---- | ------------------------------------------------- | ------------------------------------------------------------- |
| 400  | `title` / `bgmId` 缺失，或 `torrent` 解析失敗       | 檢查必填項；確認上傳的是合法 .torrent 檔案                       |
| 401  | `Authorization` 頭缺失或格式錯誤                    | `Bearer ` 字首必填，注意空格                                    |
| 403  | API Key 無效 / 已撤銷 / 不屬於該字幕組              | 重新生成 Key                                                  |
| 409  | 同一 infoHash 已經發布過                           | 想替換請用 `version=v2` 重發                                   |
| 413  | 種子檔案 / 表單超過大小限制                         | 壓縮種子（裁剪 tracker 列表）                                   |
| 429  | 寫入配額耗盡                                       | 按響應頭 `Retry-After` 退避                                    |

下一步：發完直接訂閱 [`/rss/anime.xml`](/docs/open-api/rss-anime) 驗證；找 bgmId 用 [Bangumi 搜尋](/docs/open-api/bgm-search)；想看自己字幕組的統計走 [`/api/subtitle-groups/me`](/docs/open-api/me)。
