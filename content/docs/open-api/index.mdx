AniBT 把公开数据通过两类端点暴露给第三方：用于订阅的 **RSS** 和用于程序集成的 **JSON API**。所有接口对公网匿名可读，统一支持 **HTTP 304 Not Modified**，可放心高频轮询。

<Callout type="info">
  qBittorrent / Sonarr / FlexGet / Tiny Tiny RSS 等标准 RSS 客户端**无需任何额外配置**，缓存与条件请求在响应头里自动生效。
</Callout>

## 设计逻辑

整个体系按"职责分层"组织，便于第三方理解每个端点何时该用：

```txt title="数据流"
┌────────────────────────────────────────────────────────────┐
│              第三方应用 (downloader / bot)                   │
└────────────────────────────────────────────────────────────┘
        │ ① 启动时 / 每天 1 次：拉取字幕组目录
        ▼
┌────────────────────────────────────────────────────────────┐
│  GET https://site.anibt.net/api/subtitle-groups              │
│  返回 [{slug, name, totalReleases, totalAnimes, ...}]        │
│  → 把"展示名"或"用户输入"映射到稳定 slug                      │
└────────────────────────────────────────────────────────────┘
        │ ② 用户订阅时 / 每 N 分钟：拉 RSS
        ▼
┌────────────────────────────────────────────────────────────┐
│  /rss/anime.xml?bgmId=...&groupSlug=...&resolution=...&...  │   主入口（最灵活）
│  /rss/group/<slug>.xml                                      │   字幕组短链
│  /rss/magnets.xml                                           │   全站新片
│  返回 RSS 2.0 + BEP 36（enclosure + <torrent infoHash>）    │
└────────────────────────────────────────────────────────────┘
        │ ③ 每一次轮询都带上上次的 ETag
        ▼
┌────────────────────────────────────────────────────────────┐
│  服务端用弱 ETag 快速比较：                                    │
│   - 数据未变：304 Not Modified（空 body，几十字节）            │
│   - 数据变化：200 + 完整 RSS                                 │
└────────────────────────────────────────────────────────────┘
```

## 接口稳定性承诺

- 路径与查询参数视为公开契约，**不会做不兼容变更**；新增参数只会扩展。
- `slug` 一旦发布也尽量保持稳定。
- 内部主键（如 `organizationId`）不属于公开契约，第三方一律使用 `slug`。
