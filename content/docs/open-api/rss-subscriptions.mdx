title: 个人订阅 RSS
description: GET /rss/subscriptions.xml —— 用户在前端勾选的"我的订阅"组合，输出一份个人化 RSS。需登录认证。
---

`/rss/subscriptions.xml` 把你在 AniBT 前端"我的订阅"里勾选的所有番剧 / 字幕组 / 过滤条件**合并**成一份 RSS，方便丢进 BT 客户端或 RSS 阅读器订阅。

与 [`/rss/anime.xml`](/docs/open-api/rss-anime)、[`/rss/group/<slug>.xml`](/docs/open-api/other-endpoints) 这类**对所有人公开**的 feed 不同，这是**用户私人**的 feed —— 因此需要登录认证。

<Callout type="info">
  订阅本身在前端配置：勾番剧、勾字幕组、设清晰度 / 字幕语言过滤。这个端点只负责**把当前订阅状态导出成 RSS**。
</Callout>

## 请求

```http title="HTTP"
GET https://anibt.net/rss/subscriptions.xml
```

需要**登录后**才能访问：

- 在浏览器里：登录 AniBT 账号后直接打开链接即可下载。
- 在 BT 客户端 / RSS 阅读器：从你的账户后台获取个人订阅链接（带签名参数或 token），把完整 URL 配进客户端。

具体的 token 生成 / 鉴权头格式以站点用户中心为准；运营策略可能调整，**不要硬编码**链接形态到长期运行的脚本里——优先复制后台生成的最新链接。

<Callout type="warn">
  订阅链接里通常带有用户身份凭证。**不要**把它贴到 issue / 截图 / 公共仓库 / 公共群里。泄露请立刻到后台重置。
</Callout>

## 响应

标准 RSS 2.0，结构与 [`/rss/anime.xml`](/docs/open-api/rss-anime) 完全一致：每条 `<item>` 是一次发布，附带磁力 `<enclosure>`、文件大小、字幕组、集数等。区别只在于"哪些条目能进来"由你的订阅配置决定。

```xml title="片段"
<rss version="2.0">
  <channel>
    <title>AniBT - 我的订阅</title>
    <link>https://anibt.net/subscriptions</link>
    <description>由 12 个订阅条目合并生成</description>
    <item>
      <title>[字幕组A] 番名 - 01 [1080p][CHS&JP].mkv</title>
      <link>https://anibt.net/release/...</link>
      <pubDate>...</pubDate>
      <enclosure url="magnet:?xt=urn:btih:..." length="734003200" type="application/x-bittorrent" />
    </item>
    ...
  </channel>
</rss>
```

## 缓存与延迟

- **私人内容**：响应头是 `Cache-Control: private`，CDN **不会**做共享缓存，因此你每次拉取都是当前订阅的最新结果。
- **频率**：建议 BT 客户端配 5–10 分钟轮询一次。后端单用户维度做了限流，过快轮询会被节流。
- **ETag**：返回 `ETag` + 支持 `If-None-Match` 304，减少传输量。

详见 [缓存机制](/docs/open-api/caching) 和 [限流与上限](/docs/open-api/limits)。

## 与公开 RSS 的对比

| 维度        | `/rss/anime.xml` / `/rss/group/<slug>.xml` | `/rss/subscriptions.xml`            |
| ----------- | ------------------------------------------ | ----------------------------------- |
| 鉴权        | 不需要                                     | 需要（Bearer 或 Cookie）            |
| 内容来源    | 单番剧 / 单字幕组                          | 用户在前端勾选的多条订阅合并        |
| 缓存策略    | CDN 共享缓存                                | 私有，不走共享 CDN                  |
| 适合场景    | 跟一部番、跟一个组                         | 一份 feed 涵盖你**全部**追番        |

## 常见错误

| HTTP | 原因                                      | 处理                                  |
| ---- | ----------------------------------------- | ------------------------------------- |
| 401  | 未登录 / 凭证过期                          | 重新登录，或在后台重新获取订阅链接      |
| 403  | 凭证已吊销 / 用户被封禁                    | 联系站点管理员                          |
| 429  | 轮询过快                                   | 拉长轮询间隔（建议 ≥ 5 分钟）            |
