## 调试快查

<Tabs items={['看 RSS 内容', '验证 304', '查全部 slug', '查目标字幕组']}>
  <Tab value="看 RSS 内容">
    ```bash title="前 3 个 item"
    curl -s 'https://anibt.net/rss/anime.xml?bgmId=543360&groupSlug=kirara-fantasia&limit=3' \
      | grep -E '<title>|<pubDate>'
    ```
  </Tab>
  <Tab value="验证 304">
    ```bash title="ETag 复用"
    H=$(curl -sI 'https://anibt.net/rss/anime.xml?bgmId=543360')
    echo "$H" | grep -iE 'etag|last-modified'
    ETAG=$(echo "$H" | grep -i '^etag:' | cut -d' ' -f2- | tr -d '\r')
    curl -sI -H "If-None-Match: $ETAG" 'https://anibt.net/rss/anime.xml?bgmId=543360' | head -1
    ```
  </Tab>
  <Tab value="查全部 slug">
    ```bash title="全部字幕组"
    curl -s https://site.anibt.net/api/subtitle-groups \
      | jq '.data[] | {name, slug, totalReleases}'
    ```
  </Tab>
  <Tab value="查目标字幕组">
    ```bash title="按名字过滤"
    curl -s https://site.anibt.net/api/subtitle-groups \
      | jq '.data[] | select(.name | test("Kirara Fantasia|Prejudice-Studio|北宇治字幕组|拨雪寻春")) | {name, slug}'
    ```
  </Tab>
</Tabs>

## 常见问题

<Accordions>
  <Accordion title="slug 和 organizationId 有什么区别？">
    `slug` 是公开 URL 用的稳定字符串；`organizationId` 是内部主键。**第三方一律用 `slug`。**
  </Accordion>
  <Accordion title="为什么我的 If-Modified-Since 没生效？">
    按 RFC 7232，**ETag（`If-None-Match`）优先**。如果你同时发了两个头，服务端只看 ETag。建议永远用 ETag。
  </Accordion>
  <Accordion title="episode=5 会匹配 episodeKey='05' 吗？">
    是的，二者都会过滤；但 `episode` 是数字精确匹配，`episodeKey` 是字符串原样匹配。合集请用 `episodeKey=BATCH`。
  </Accordion>
  <Accordion title="我想知道某个 release 的种子直链怎么办？">
    RSS 里 `<enclosure url>` 已经是可直接喂给 BT 客户端的磁力 URL（带默认 tracker），`<torrent>` 元素里也带 infoHash。
  </Accordion>
  <Accordion title="接口会变吗？">
    路径与查询参数视为公开契约，**不会做不兼容变更**；新增参数只会扩展。`slug` 一旦发布也尽量保持稳定。
  </Accordion>
  <Accordion title="为什么我配置了极严格的过滤后返回数量很少？">
    `/rss/anime.xml` 的候选扫描上限是 4000 条，过滤极严格时（例如某番剧某组只有几集 1080p 简中外挂）后端可能在 4000 条候选里筛不到很多。这是设计上的安全阀，详见 [限流与上限](/docs/open-api/limits)。
  </Accordion>
  <Accordion title="新发的种子多久能在 RSS 里看到？">
    取决于你客户端的轮询间隔与 CDN 缓存窗口的叠加。典型 5–30 分钟，详见 [限流与上限](/docs/open-api/limits)。需要实时通知请用 webhook / Telegram bot。
  </Accordion>
</Accordions>

## 接入 checklist

最后过一遍：

<Steps>
  <Step>
    ### 启动时拉一次 [`/api/subtitle-groups`](/docs/open-api/subtitle-groups)
    本地缓存 `name → slug` 映射，避免硬编码 slug。
  </Step>
  <Step>
    ### 用 [`/rss/anime.xml`](/docs/open-api/rss-anime) 或短链订阅
    优先 `bgmId + groupSlug` 组合；不需要过滤时用 `/rss/group/<slug>.xml`。
  </Step>
  <Step>
    ### 永远带 If-None-Match
    缓存上次响应的 `ETag`，下次请求带 `If-None-Match`，详见 [HTTP 缓存与 304](/docs/open-api/caching)。
  </Step>
  <Step>
    ### 间隔 ≥ 5 分钟
    更短没收益，且容易触发 429。
  </Step>
  <Step>
    ### 见 429 用 Retry-After 退避
    不要立即重试。
  </Step>
</Steps>
