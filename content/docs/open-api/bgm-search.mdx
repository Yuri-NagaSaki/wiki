`/api/bgm/search` 是**公开**搜索端点，无需鉴权。用于把"用户输入的番剧名"映射成稳定的 Bangumi.tv subject id（`bgmId`），后续就能喂给 [`/rss/anime.xml`](/docs/open-api/rss-anime) 或 [`/api/releases/publish`](/docs/open-api/releases-publish)。

## 请求

```txt title="URL"
GET https://site.anibt.net/api/bgm/search?q=<keyword>&limit=<n>
```

支持中文、英文、日文、罗马音任意关键词。

```bash title="curl 示例"
curl -s -G 'https://site.anibt.net/api/bgm/search' \
  --data-urlencode 'q=葬送的芙莉莲' -d 'limit=3'
```

## 实战：搜索 → 拼 RSS

```bash title="一行打出 bgmId 给 RSS 用"
BGM=$(curl -s -G 'https://site.anibt.net/api/bgm/search' \
  --data-urlencode 'q=葬送的芙莉莲' -d 'limit=1' \
  | jq -r '.data[0].bgmId')

echo "https://anibt.net/rss/anime.xml?bgmId=$BGM&resolution=1080p&language=CHS"
```

## 注意事项

<Callout type="info">
  搜索结果按 Bangumi 内部相关度排序，**不**保证第一条就是最热门版本。如果番名容易撞车（例如有多季 / 同名作品），建议把 `limit` 设大一点，再人工挑选。
</Callout>

<Callout type="warn">
  这是**外部数据透传**接口，结果质量直接受 Bangumi 上游影响。生产代码里建议把 `bgmId` 缓存到本地，不要每次都查。
</Callout>
