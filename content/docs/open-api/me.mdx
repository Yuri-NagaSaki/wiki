---
title: 字幕组自身信息
description: GET /api/subtitle-groups/me —— 用 API Key 查询当前字幕组的基本信息与统计数据。
---

`/api/subtitle-groups/me` 是**鉴权读取**端点。带上字幕组 API Key，返回该字幕组的基本信息（名称、slug、简介、网站）和统计数据（发布数、覆盖番剧数、成员数）。

适合用在：CI 启动时打印当前用哪把 Key 跑、控制台 dashboard 展示字幕组当前状态、自动化脚本里做"是不是登在正确字幕组"的最小校验。

## 请求

```http title="HTTP"
GET https://site.anibt.net/api/subtitle-groups/me
Authorization: Bearer <YOUR_API_KEY>
```

```bash title="curl 示例"
curl -H "Authorization: Bearer <YOUR_API_KEY>" \
  https://site.anibt.net/api/subtitle-groups/me
```

## 响应

```jsonc title="200 OK"
{
  "ok": true,
  "data": {
    "_id": "...",
    "name": "My Group",
    "slug": "my-group",
    "description": "...",
    "website": "https://...",
    "stats": {
      "totalReleases": 42,
      "totalAnimes": 10,
      "memberCount": 5
    },
    "status": "active",
    "createdAt": 1700000000000
  }
}
```

## 与公共目录的关系

<Callout type="info">
  与 [`/api/subtitle-groups`](/docs/open-api/subtitle-groups) 的核心差别：
  - `/api/subtitle-groups`：**公开**，返回**所有**字幕组的精简信息，不需要鉴权。
  - `/api/subtitle-groups/me`：**鉴权**，只返回**调用方**这一组的详细信息，附带 `stats` 与内部 `_id`、`description`、`status`。
</Callout>

## 错误响应

| HTTP | 原因                            | 处理                            |
| ---- | ------------------------------- | ------------------------------- |
| 401  | `Authorization` 头缺失或格式错误 | `Bearer ` 前缀必填，注意空格      |
| 403  | API Key 无效 / 已撤销           | 在后台重新生成                  |
