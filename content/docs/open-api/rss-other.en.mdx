---
title: Other release RSS
description: GET /rss/other/<category>.xml — separate RSS feeds for manga, music, RAW, and stage releases; not mixed into the main RSS.
---

`/rss/other/<category>.xml` exports dedicated RSS feeds for other release categories. The supported categories are `manga`, `music`, `raw`, and `stage`.

<Callout type="warn">
  Other releases are **not** mixed into `/rss/magnets.xml`, `/rss/anime.xml`, or anime detail RSS feeds. Subscribe to the category feeds on this page when you need them.
</Callout>

## URLs

```txt title="Category RSS"
https://anibt.net/rss/other/manga.xml
https://anibt.net/rss/other/music.xml
https://anibt.net/rss/other/raw.xml
https://anibt.net/rss/other/stage.xml
```

## Examples

<Steps>
  <Step>
    ### Latest manga releases
    ```txt title="Manga"
    https://anibt.net/rss/other/manga.xml
    ```
  </Step>
  <Step>
    ### Search music releases
    ```txt title="Music + keyword"
    https://anibt.net/rss/other/music.xml?q=FLAC&limit=50
    ```
  </Step>
  <Step>
    ### Sort RAW by file size
    ```txt title="RAW + file size"
    https://anibt.net/rss/other/raw.xml?sortField=fileSize&sortOrder=desc
    ```
  </Step>
</Steps>

## Cache and indexing

- Responses support `ETag` / `If-None-Match`; unchanged feeds return `304 Not Modified`.
- Shared cache uses `s-maxage=120` with long `stale-while-revalidate` / `stale-if-error` windows.
- These RSS responses use a `noindex, follow` search engine policy so category feeds are not indexed.
