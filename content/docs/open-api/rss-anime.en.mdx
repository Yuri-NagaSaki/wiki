`/rss/anime.xml` is the most flexible subscription endpoint: combine `bgmId` + `groupSlug` + multiple tags (`resolution` / `language` / `format` / `subtitle` / `episode` / `episodeKey`) in any way to subscribe to a show, a group, or the whole site.

<Callout type="warn">
  At least one of `bgmId` or `groupSlug` is required. Requests with neither are rejected.
</Callout>

## URL

```txt title="URL"
GET https://anibt.net/rss/anime.xml?<query>
```

## Two equivalent ways to repeat a parameter

```txt title="Standard repeated query (recommended)"
?resolution=1080p&resolution=720p
```

```txt title="Comma-separated (also accepted)"
?resolution=1080p,720p
```

## How the backend chooses an index

The server picks the most efficient index based on the query you provide:

| Provided                | Index used               |
| ----------------------- | ------------------------ |
| `bgmId` + `groupSlug`   | `groupId_animeId`        |
| `bgmId` only            | `animeId`                |
| `groupSlug` only        | `groupId_publishedAt`    |

Then it runs an in-memory tag filter (**up to 4000 candidate rows**), sorts by `publishedAt` desc, and truncates to `limit`. The SSR layer adds **300 seconds of caching** (keyed on the canonicalised query string), so a given query hits the backend at most once per cache window.

## Mikan compatibility

Old Mikan links can switch host and keep working as-is:

<Tabs items={['Original Mikan link', 'Switch host only', 'Recommended']}>
  <Tab value="Original Mikan link">
    ```txt
    https://mikanani.me/RSS/Bangumi?bangumiId=3921&subgroupid=615
    ```
  </Tab>
  <Tab value="Switch host only">
    ```txt
    https://anibt.net/rss/anime.xml?bangumiId=3921&subgroupid=615
    ```
    `bangumiId` and `subgroupid` are accepted as compatibility aliases.
  </Tab>
  <Tab value="Recommended">

  </Tab>
</Tabs>
