AniBT 不只是一个站。下面这些项目围绕"字幕组发布资源"这件事，分别解决**字体太大**和**做种分散**两个真实痛点，配合主站使用最顺手。

## FontInAss

<ProjectCard
  owner="Yuri-NagaSaki"
  repo="FontInAss"
  description="字幕字体子集化处理与分享工具：扫描 ASS / SRT 用到的字符，从原字体里裁出最小可用子集，打包随片分发，体积可压到原字体的 1–5%。"
  links={[
    { label: 'font.anibt.net 在线使用', href: 'https://font.anibt.net' },
    { label: 'GitHub 仓库', href: 'https://github.com/Yuri-NagaSaki/FontInAss' },
  ]}
/>

**为什么需要子集化**

中日字体动辄 10–30 MB，一部 12 集的番压完字幕字体能比片源还大。子集化扫出 ASS 实际用到的字符，只保留这些字形，常见可压到 100–500 KB。FontInAss 把这一步从命令行批处理变成可视化流程，并且**记录哪些字符被裁掉**，方便后续追加补字。

**典型流程**

<Steps>
  <Step>
    ### 上传 ASS / SRT 与原字体
    支持批量字幕、多字体；自动识别 `\fn` 字段。
  </Step>
  <Step>
    ### 子集化
    工具裁出每个字体的最小子集，输出 `.ttf` / `.otf`。
  </Step>
  <Step>
    ### 内嵌发布
    把子集字体放进 `Fonts/` 子目录，与字幕一起打进种子；播放器加载即可正确渲染，无需观众安装字体。
  </Step>
</Steps>

## AniBT-Speed

<ProjectCard
  owner="Yuri-NagaSaki"
  repo="AniBT-Speed"
  description="动漫 BT 的自动做种 / 分流工具：拉取 RSS、自动下载、长期做种，给热门资源补充上传带宽。"
  links={[
    { label: 'GitHub 仓库', href: 'https://github.com/Yuri-NagaSaki/AniBT-Speed' },
  ]}
/>

**它解决什么**

字幕组发布后头几天的热度爆点，往往会出现"种子在但上传带宽不够"的现象。AniBT-Speed 是一个**自动做种节点**程序：

- 订阅 [`/rss/anime.xml`](/docs/open-api/rss-anime) 或 [`/rss/magnets.xml`](/docs/open-api/other-endpoints)。
- 自动下载新发布。
- **长期做种**，不主动删除，给后来者提供上传带宽。

适合手里有富余带宽的爱好者参与建设公共做种生态。
