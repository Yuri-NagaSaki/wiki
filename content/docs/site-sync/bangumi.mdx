萌番组（bangumi.moe）使用登录 Cookie 鉴权，需要从 DevTools 抓取登录请求里的 Cookie 整段复制。

<Callout type="warn">
  必须**先退出登录、清空 Network 后再重新登录**，才能抓到完整的登录 Cookie。
</Callout>

## 步骤一：开启 Network 录制

<Steps>
  <Step>
    ### 退出登录并刷新

    打开 [https://bangumi.moe](https://bangumi.moe)，先 **退出登录** 然后 **刷新网页**。
  </Step>

  <Step>
    ### 开启网络录制

    按 <kbd>F12</kbd> 打开 DevTools，切到 **Network（网络）** 页。确认录制按钮为 **红色**（已开启录制）。

    ![DevTools Network 录制网络日志](/images/site-sync/bangumi-network-recording.webp)
  </Step>

  <Step>
    ### 重新登录

    输入账号密码，点击登录。
  </Step>
</Steps>

## 步骤二：复制 Cookie

<Steps>
  <Step>
    ### 找到 disqus 请求

    在 **Network** 页中找到名为 **`disqus`** 的请求，点击它。

    ![Network 中的 disqus 请求](/images/site-sync/bangumi-disqus-request.webp)
  </Step>

  <Step>
    ### 复制 Request Headers 里的 Cookie

    右侧切到 **Headers（标头）** 页，下拉找到 **Request Headers** → **Cookie**，全选这一长串。

    <Callout type="warn">
      千万**不要选到下方** `priority` 的 `u=1, i`。
    </Callout>

    ![复制 Request Headers 里的 Cookie](/images/site-sync/bangumi-cookie-header.webp)
  </Step>

  <Step>
    ### 粘贴到 AniBT 并保存

    回到 AniBT 粘贴 Cookie，点击保存。

    ![在 AniBT 中粘贴 Cookie](/images/site-sync/bangumi-anibt-cookie-save.webp)
  </Step>
</Steps>

## 测试连接

填写完后点击 **保存并测试连接**：

- **连接成功**：配置完成。
- **连接失败**：检查 Cookie 是否复制完整、账号是否登录成功。

![测试连接成功](/images/site-sync/bangumi-connection-success.webp)
