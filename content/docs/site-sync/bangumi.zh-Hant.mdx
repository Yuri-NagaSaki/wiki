title: 萌番組同步

---

萌番組（bangumi.moe）使用登入 Cookie 鑑權，需要從 DevTools 抓取登入請求裡的 Cookie 整段複製。

<Callout type="warn">
  必須**先登出、清空 Network 後再重新登入**，才能抓到完整的登入 Cookie。
</Callout>

## 步驟一：開啟 Network 錄製

<Steps>
  <Step>
    ### 登出並重新整理

    開啟 [https://bangumi.moe](https://bangumi.moe)，先 **登出** 然後 **重新整理網頁**。
  </Step>

  <Step>
    ### 開啟網路錄製

    按 <kbd>F12</kbd> 開啟 DevTools，切到 **Network（網路）** 頁。確認錄製按鈕為 **紅色**（已開啟錄製）。

    ![DevTools Network 錄製網路日誌](/images/site-sync/bangumi-network-recording.webp)
  </Step>

  <Step>
    ### 重新登入

    輸入帳號密碼，點選登入。
  </Step>
</Steps>

## 步驟二：複製 Cookie

<Steps>
  <Step>
    ### 找到 disqus 請求

    在 **Network** 頁中找到名為 **`disqus`** 的請求，點選它。

    ![Network 中的 disqus 請求](/images/site-sync/bangumi-disqus-request.webp)
  </Step>

  <Step>
    ### 複製 Request Headers 裡的 Cookie

    右側切到 **Headers（標頭）** 頁，下拉找到 **Request Headers** → **Cookie**，全選這一長串。

    <Callout type="warn">
      千萬**不要選到下方** `priority` 的 `u=1, i`。
    </Callout>

    ![複製 Request Headers 裡的 Cookie](/images/site-sync/bangumi-cookie-header.webp)
  </Step>

  <Step>
    ### 貼上到 AniBT 並儲存

    回到 AniBT 貼上 Cookie，點選儲存。

    ![在 AniBT 中貼上 Cookie](/images/site-sync/bangumi-anibt-cookie-save.webp)
  </Step>
</Steps>

## 測試連線

填寫完後點選 **儲存並測試連線**：

- **連線成功**：設定完成。
- **連線失敗**：檢查 Cookie 是否複製完整、帳號是否登入成功。

![測試連線成功](/images/site-sync/bangumi-connection-success.webp)
